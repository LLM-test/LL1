# День 12: Персонализация ассистента

## Задание

Реализовать профиль пользователя, который автоматически влияет на поведение агента: уровень экспертизы, стиль и формат ответов, язык, ограничения. Без изменения базового промпта и без ручной настройки при каждом запросе.

---

## Зачем нужны профили при создании агента

### Одна модель — разные аудитории

Базовый system prompt задаёт **что умеет** агент. Профиль задаёт **для кого** он работает. Один и тот же агент с одним и тем же LLM может давать принципиально разные ответы на один вопрос — в зависимости от того, кто спрашивает:

- Junior-разработчику нужны пошаговые объяснения и примеры
- Senior-разработчику — только суть без воды
- Студенту — академический стиль с терминологией

Без профиля агент отвечает «в среднем» — неоптимально для всех сразу.

### Профиль vs. Память — две разные вещи

| | Профиль | Память |
|---|---|---|
| **Что хранит** | Кто такой пользователь | Что происходило в диалогах |
| **Как формируется** | Вручную, один раз | Автоматически, из разговоров |
| **Когда меняется** | Редко (смена роли/цели) | Постоянно |
| **Вопрос** | «Кто спрашивает?» | «О чём говорили?» |

Это два независимых измерения контекста. Агент знает и **кто** ты, и **что** происходит — и использует оба источника при каждом запросе.

### Профиль убирает необходимость повторяться

Без профиля пользователь вынужден объяснять контекст заново в каждом новом диалоге: «я джун, объясни подробно», «отвечай только на русском», «не используй паттерны, которых я ещё не знаю». С профилем это вшито в агента один раз и работает всегда — прозрачно для пользователя.

### Constraints — инструмент управления форматом

Поле `constraints` — самое мощное. Оно позволяет не только менять стиль, но и жёстко управлять форматом вывода:

- `"отвечай только кодом без объяснений"` → агент перестаёт писать текст
- `"не используй паттерны выше уровня Junior"` → агент упрощает решения
- `"всегда давай два варианта: простой и оптимальный"` → агент меняет структуру каждого ответа

Это дешевле и надёжнее, чем пытаться управлять форматом через промпт в каждом сообщении.

### Порядок инжекции имеет значение

```
[базовый system prompt]   ← что агент умеет
[профиль пользователя]    ← кто спрашивает
[память агента]           ← что происходит сейчас
```

Профиль идёт раньше памяти: он задаёт «линзу», через которую агент интерпретирует контекст задачи. Если сначала описать задачу, а потом сказать «ты джун» — модель уже сформировала образ ответа без учёта аудитории.

### Пресеты — способ онбординга пользователей

Готовые пресеты (Junior/Senior/Student) решают проблему пустого листа: пользователь не знает, что писать в полях профиля. Пресет заполняет разумные дефолты, которые сразу работают. Пользователь может скорректировать детали — но основа уже есть.

В продакшн-агентах это особенно важно: первые минуты после установки определяют, останется ли пользователь.

---

## Вывод

Профиль — это способ сделать одного агента полезным для разных людей, не обучая отдельную модель для каждого. Вся персонализация происходит на уровне контекста, который агент получает при каждом запросе. Пользователь настраивает профиль один раз — и агент адаптируется автоматически, без изменения кода и без ручных инструкций в каждом сообщении.
